# JSONTool 产品需求与变更记录（Cursor 规则文件）

## 目的

把“需求、约束、决策”放在**项目内**，让后续任何改动都有依据，避免口头需求丢失导致返工。

## 记录规则（重要）

1. 只记录**用户明确提出**的需求/约束/偏好，不写推测。
2. 每条需求必须可执行，尽量一句话说清。
3. 每次用户提出新要求时，都要**同步更新本文件**（这条规则本身也属于需求的一部分）。
4. 变更要写“从什么变成什么”，并注明日期（YYYY-MM-DD）。

## 已确认需求（持续更新）

- **入口**：独立页面（非 popup），点击扩展图标打开。
- **核心功能**：JSON 格式化、对比、表格化、排序、历史缓存（本机）。
- **多窗口/多文档**：一个主窗口，顶部 Tab，点击 `+` 新增 JSON 窗口。
- **编辑体验**：
  - 单编辑器模式：格式化/压缩/排序等操作直接作用于当前编辑器内容（原地修改）。
  - 编辑器需支持语法高亮（类似代码编辑器）。
- **主题**：支持亮色/暗色切换。
- **样式工程**：全站样式使用 SCSS 实现。使用 Fluent 2 的设计风格
